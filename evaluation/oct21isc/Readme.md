# Dataset: October 2021 ISC Forensic Contest

**Source:** SANS Internet Storm Center (ISC)  
**Link:** https://isc.sans.edu/diary/October+2021+Forensic+Contest+Answers+and+Analysis/27998  
**Size:** 46.7 MB | 7,278 Security Onion logs

## Description

A multi-host infection scenario from October 2021 involving three simultaneously compromised Windows
clients within the same Active Directory environment, each infected via a distinct malicious email. The
infections span three malware families: NanoCore RAT establishing C2 over a dynamic DNS domain, BazarLoader
delivered via a Contact Forms campaign retrieving a malicious DLL, and Qakbot distributed through a
macro-enabled Excel spreadsheet that downloads three DLL copies before generating spambot SMTP traffic.

## Repository Contents

| Folder | Contents |
|--------|----------|
| `cache_embeddings/` | Pickled FAISS embeddings generated from ES query results using `all-mpnet-base-v2` |
| `config/` | `network.txt` — LAN topology and host info; `questions.txt` — forensic questions posed to the LLM; `answers.txt` — reference answers for evaluation |
| `es_results/` | Elasticsearch query outputs from the IOC query library, targeting Suricata alerts, Zeek logs, Kerberos events, and other network indicators |
| `llm_reports/` | Incident reports generated by all evaluated LLMs (Claude, DeepSeek, GPT variants, Llama, Cisco) at the optimal chunk size (k=7) |
| `llm_reports_k1/` `llm_reports_k3/` `llm_reports_k5/` `llm_reports_k14/` | Incident reports at varying retrieval chunk sizes (k=1, 3, 5, 14), used for context sensitivity ablation |
