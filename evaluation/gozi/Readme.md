# Dataset: Gozi (March 2023)

**Source:** Palo Alto Unit 42  
**Link:** https://unit42.paloaltonetworks.com/march-wireshark-gozi-answers/  
**Size:** 5.67 MB | 897 Security Onion logs

## Description

A Gozi (ISFB/Ursnif) banking trojan infection from March 2023, initiated via a malicious ZIP archive
downloaded from a compromised site, containing an internet shortcut that retrieves the Gozi payload over
SMB. Post-infection traffic consists of encoded HTTP GET and POST requests to multiple C2 servers using
direct IP addresses rather than domains, with additional requests for encrypted plugin modules (.rar)
used for credential theft from browser caches.

## Repository Contents

| Folder | Contents |
|--------|----------|
| `cache_embeddings/` | Pickled FAISS embeddings generated from ES query results using `all-mpnet-base-v2` |
| `config/` | `network.txt` — LAN topology and host info; `questions.txt` — forensic questions posed to the LLM; `answers.txt` — reference answers for evaluation |
| `es_results/` | Elasticsearch query outputs from the IOC query library, targeting Suricata alerts, Zeek logs, Kerberos events, and other network indicators |
| `llm_reports/` | Incident reports generated by all evaluated LLMs (Claude, DeepSeek, GPT variants, Llama, Cisco) at the optimal chunk size (k=7) |
| `llm_reports_k1/` `llm_reports_k3/` `llm_reports_k5/` `llm_reports_k14/` | Incident reports at varying retrieval chunk sizes (k=1, 3, 5, 14), used for context sensitivity ablation |
