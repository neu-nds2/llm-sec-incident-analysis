# Dataset: June 2021 ISC Forensic Contest

**Source:** SANS Internet Storm Center (ISC)  
**Link:** https://isc.sans.edu/diary/June+2021+Forensic+Contest+Answers+and+Analysis/27582  
**Size:** 43.1 MB | 47,991 Security Onion logs

## Description

A multi-host infection scenario from June 2021 involving three simultaneously compromised Windows clients
within the same Active Directory environment. The infections span three distinct malware families: AgentTesla
exfiltrating credentials via SMTP, Hancitor delivering Cobalt Strike and Ficker Stealer as follow-up payloads,
and Qakbot establishing HTTPS-based C2 communication after retrieving a malicious ZIP archive. This is the
largest and most complex scenario in the dataset, making it a strong benchmark for multi-source correlation.

## Repository Contents

| Folder | Contents |
|--------|----------|
| `cache_embeddings/` | Pickled FAISS embeddings generated from ES query results using `all-mpnet-base-v2` |
| `config/` | `network.txt` — LAN topology and host info; `questions.txt` — forensic questions posed to the LLM; `answers.txt` — reference answers for evaluation |
| `es_results/` | Elasticsearch query outputs from the IOC query library, targeting Suricata alerts, Zeek logs, Kerberos events, and other network indicators |
| `llm_reports/` | Incident reports generated by all evaluated LLMs (Claude, DeepSeek, GPT variants, Llama, Cisco) at the optimal chunk size (k=7) |
| `llm_reports_k1/` `llm_reports_k3/` `llm_reports_k5/` `llm_reports_k14/` | Incident reports at varying retrieval chunk sizes (k=1, 3, 5, 14), used for context sensitivity ablation |
